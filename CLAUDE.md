# Career Shift — лендинг

## Что это
Одностраничный лендинг программы осознанной смены карьеры (career-shift). Цель — конвертировать посетителя в бесплатный 30-минутный созвон через mailto или Telegram.

## Технологии
- HTML5 (один файл `index.html`)
- Tailwind CSS — подключён через CDN (`https://cdn.tailwindcss.com`), без сборщика и npm
- Минимум JS (один скрипт для года в футере)

## Структура
- `index.html` — вся страница: header, hero, «для кого», «что получишь», «как проходит», FAQ, CTA, footer
- `памятки/` — заметки, идеи правок, скриншоты по ходу работы
- `README.md` — короткая инструкция запуска

## Как запустить
Открыть `index.html` в браузере — двойной клик или `open index.html` в терминале. Сборка не нужна.

## Особенности
- **Tailwind через CDN** — это сознательное решение для лендинга-визитки. Если страница начнёт расти (несколько HTML-файлов, кастомные плагины, оптимизация бандла), стоит перейти на сборку через PostCSS / Tailwind CLI. До этого момента CDN — правильный выбор: ноль инфраструктуры.
- **Формы нет** — вместо неё `mailto:` и ссылка на Telegram. Если потребуется форма с полями — добавлять придётся внешний сервис (Formspree, Getform) или бэкенд, чего сейчас намеренно избегаем.
- **Плейсхолдеры помечены `<!-- TODO -->`** — email, Telegram, цена, имя автора. Перед публикацией пройтись по ним grep'ом: `grep -n "TODO" index.html`.
- **Всё на русском** — целевая аудитория русскоязычная.
- **Секций немного и они короткие** — это лендинг, а не лонгрид. При правках сохранять плотность контента, не разводить простыни текста.

## Что НЕ нужно делать без обсуждения
- Не подключать сборщики (Vite, webpack, npm-пакеты) — это сломает «открыл и работает».
- Не выносить стили в отдельный CSS-файл — пока всё в одном HTML, это удобно править.
- Не добавлять формы с обработкой на фронте — без бэкенда они всё равно не сработают.
